## 핵심 원칙

### 1. 렌더링 모드 규칙

이 프로젝트는 **앱별로 다른 렌더링 모드**를 사용합니다:

| 앱 | 모드 | 데이터 소스 |
|:---|:-----|:-----------|
| **Context** | **SSR + D1** | Cloudflare D1 |
| **Permissive** | SSR | In-memory |
| **Roots** | SSG | TypeScript |

**모든 앱 공통 금지:**

- SPA 모드 (SEO 불가)
- 동적 라우트에서 `loader()` 제거

**Context 앱 (SSR + D1):**

- react-router.config.ts에서 `ssr: false` 설정 금지
- D1 없이 entry 데이터 로딩 금지

### 2. 다운그레이드 금지

문제 해결을 위해 패키지 버전을 절대 다운그레이드하지 마세요. 대신:
- 업그레이드 경로 찾기
- 호환성을 위해 코드 수정
- `pnpm.overrides`로 충돌 격리

### 3. 완전한 코드만

항상 완전하고 작동하는 코드를 제공하세요. 절대 하지 말 것:
- `// ... existing code ...` 사용
- 구현 없이 TODO 주석만 남기기
- 설명 없이 `@ts-ignore` 사용

## 응답 규칙

| 규칙 | 설명 |
|------|-------------|
| **한글 응답** | 모든 설명과 문서는 한글로 |
| **확인 후 주장** | 추측하지 말고 코드 확인 |
| **출처 명시** | `파일:라인` 형식으로 참조 |
| **완전한 코드** | `// ...` 플레이스홀더 금지 |

## 품질 우선순위

규칙이 충돌할 때 이 순서를 따르세요:

```
1. 접근성 > 2. 보안 > 3. 기능 > 4. 성능 > 5. SEO > 6. 코드 품질
```

## 변경 전 확인사항

스스로에게 물어보세요:

1. 근본 원인(WHY)을 이해했는가?
2. 기존 기능이 유지되는가?
3. 하드코딩된 값이 있는가?
4. 모든 유사한 경우에 작동하는가?

## 토큰 효율성

### 모범 사례

| 작업 유형 | 방법 | 이유 |
|----------|--------|-----|
| 코드베이스 탐색 | Task tool (Explore) | 서브에이전트가 탐색, 요약 반환 |
| 단순 파일 검색 | Task tool + haiku | 빠른 검색을 위한 저비용 모델 |
| 파일 편집 | 직접 Read + Edit | 정확한 파일만 처리 |
| 빌드/테스트 로그 | `head -50` 요약 | 전체 로그 출력 방지 |

### 하지 말 것

- 전체 탐색을 위한 Glob → Read 반복
- 전체 빌드 로그 출력
- 같은 파일 여러 번 읽기
