## 核心原則

### 1. SSG専用 — 例外なし

このプロジェクトは**100%静的サイト生成**を使用しています。絶対にしないこと：
- `ssr: true`設定
- `prerender()`関数の削除
- 空の`<div id="root"></div>`でSPA作成

### 2. ダウングレード禁止

問題解決のためにパッケージバージョンを絶対にダウングレードしないでください。代わりに：
- アップグレードパスを探す
- 互換性のためにコードを修正
- `pnpm.overrides`で競合を分離

### 3. 完全なコードのみ

常に完全で動作するコードを提供してください。絶対にしないこと：
- `// ... existing code ...`の使用
- 実装なしでTODOコメントだけ残す
- 説明なしで`@ts-ignore`使用

## レスポンスルール

| ルール | 説明 |
|------|-------------|
| **韓国語レスポンス** | すべての説明とドキュメントは韓国語で |
| **確認後主張** | 推測せずコード確認 |
| **出典明示** | `ファイル:行`形式で参照 |
| **完全なコード** | `// ...`プレースホルダー禁止 |

## 品質優先順位

ルールが競合する場合、この順序に従ってください：

```
1. アクセシビリティ > 2. セキュリティ > 3. 機能 > 4. パフォーマンス > 5. SEO > 6. コード品質
```

## 変更前の確認事項

自分に問いかけてください：

1. 根本原因（WHY）を理解したか？
2. 既存機能が維持されるか？
3. ハードコードされた値があるか？
4. すべての類似ケースで動作するか？

## トークン効率

### ベストプラクティス

| タスクタイプ | 方法 | 理由 |
|----------|--------|-----|
| コードベース探索 | Task tool (Explore) | サブエージェントが探索、サマリー返却 |
| 単純ファイル検索 | Task tool + haiku | 高速検索のための低コストモデル |
| ファイル編集 | 直接Read + Edit | 正確なファイルのみ処理 |
| ビルド/テストログ | `head -50`サマリー | 全ログ出力防止 |

### してはいけないこと

- 全体探索のためのGlob → Read繰り返し
- 全ビルドログ出力
- 同じファイルを複数回読む
