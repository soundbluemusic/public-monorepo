# 기술 문서 사이트 리디자인 보고서

## 현재 상태 분석

### 현재 디자인의 문제점

| 구분 | 현재 상태 | 문제점 |
|------|-----------|--------|
| **레이아웃** | 중앙 정렬, 히어로 섹션 중심 | 마케팅 랜딩 페이지 느낌 |
| **네비게이션** | 상단 헤더만 존재 | 문서 탐색이 어려움 |
| **콘텐츠 구조** | 카드 그리드 형태 | 문서 계층 구조 부재 |
| **타이포그래피** | 큰 제목, 짧은 설명 | 기술 문서에 적합하지 않음 |
| **정보 밀도** | 낮음 (여백 과다) | 효율적인 정보 전달 어려움 |

### 현재 페이지 구성 (index.tsx)
```
┌─────────────────────────────────────┐
│           상단 헤더 (Sticky)         │
├─────────────────────────────────────┤
│                                     │
│         🎯 히어로 섹션               │
│     (큰 타이틀 + 검색바)             │
│                                     │
├─────────────────────────────────────┤
│   [카드1]   [카드2]   [카드3]        │
│   웹 API   오픈소스    순위          │
├─────────────────────────────────────┤
│                                     │
│        순위 프리뷰 (3열 카드)        │
│                                     │
├─────────────────────────────────────┤
│              푸터                    │
└─────────────────────────────────────┘
```

## 제안하는 리디자인

### 1. 레이아웃 변경: 3열 문서 레이아웃

**변경 전 (현재)**
```
[          중앙 정렬 콘텐츠           ]
```

**변경 후 (문서 스타일)**
```
┌────────┬─────────────────────┬────────┐
│사이드바 │     메인 콘텐츠      │  ToC   │
│(240px) │     (유동폭)        │(200px) │
│        │                     │        │
│ 📂 섹션1│                     │ 현재   │
│  ├ 문서 │                     │ 페이지 │
│  └ 문서 │                     │ 목차   │
│ 📂 섹션2│                     │        │
│  ├ 문서 │                     │        │
│  └ 문서 │                     │        │
└────────┴─────────────────────┴────────┘
```

### 2. 주요 UI 컴포넌트

#### A. 사이드바 네비게이션
```tsx
// 계층적 문서 구조
- 시작하기
  ├ 소개
  └ 빠른 시작
- 웹 표준 API
  ├ DOM API
  ├ Fetch API
  ├ Web Storage
  └ ... (50+ APIs)
- 오픈소스 라이브러리
  ├ 프레임워크
  ├ 상태 관리
  ├ 라우팅
  └ ... (카테고리별)
- 순위
  ├ 인기순
  ├ 경량순
  └ 급상승
```

#### B. 목차 (Table of Contents)
- 현재 페이지의 헤딩(h2, h3) 기반
- 스크롤에 따라 현재 위치 하이라이트
- 클릭 시 해당 섹션으로 스크롤

#### C. 빵부스러기 네비게이션
```
홈 > 오픈소스 > 프레임워크 > Solid.js
```

#### D. 이전/다음 페이지 네비게이션
```
┌─────────────────┬─────────────────┐
│ ← 이전          │          다음 → │
│ Fetch API       │ Web Storage     │
└─────────────────┴─────────────────┘
```

### 3. 타이포그래피 개선

| 요소 | 현재 | 제안 |
|------|------|------|
| **H1** | 4xl-6xl, 중앙 | 3xl, 좌측 정렬 |
| **H2** | 2xl, 중앙 | xl, 좌측, 경계선 |
| **본문** | 큰 여백, 카드 | 컴팩트, 문단 중심 |
| **코드** | 없음 | 신택스 하이라이팅 |

### 4. 색상 스키마 조정

```
현재: 밝은 그린 primary (#22c55e) - 마케팅 느낌
제안: 차분한 블루/슬레이트 - 문서 사이트 느낌

문서 사이트 색상 예시:
- 사이드바 배경: slate-50 / dark:slate-900
- 활성 메뉴: blue-500 좌측 보더
- 본문 링크: blue-600
- 코드 블록: slate-800 / dark:slate-950
```

```
┌─────────────────────────────────────┐
│  Permissive        웹API 오픈소스.. │  ← 심플 헤더
├─────────────────────────────────────┤
│                                     │
│           [Permissive]              │  ← 큰 히어로

└─────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────┐
│  📚 Permissive Docs    [🔍 검색...]    [🌙/☀️]  │
├─────────┬───────────────────────────────┬───────┤
│         │ 홈 > 웹 API > Fetch API       │       │
│시작하기 │                               │ 목차  │

│ ▸ ...   │ Fetch API는 네트워크...       │       │
│         │                               │       │

│         │                               │       │
│순위     │ ## 브라우저 호환성            │       │

## 결론

현재 사이트는 **마케팅/랜딩 페이지** 스타일로 설계되어 있습니다. 기술 문서 사이트로 전환하려면:

1. **3열 레이아웃** 도입 (사이드바 - 콘텐츠 - 목차)
2. **계층적 네비게이션** 구조 추가
3. **컴팩트한 타이포그래피**로 정보 밀도 향상
4. **문서 탐색 도구** 추가 (빵부스러기, 이전/다음)
5. **색상 톤** 조정 (마케팅 그린 → 문서 블루/슬레이트)

이 변경을 통해 사용자가 원하는 정보를 빠르게 찾고, 체계적으로 학습할 수 있는 기술 문서 사이트로 전환됩니다.
